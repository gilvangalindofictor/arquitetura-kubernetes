# Skill: Brainstorm Técnico
Método divergente → convergente. Regra: não tomar decisões, apenas gerar material e opções.

Processo:
1. Levantar 3-5 opções arquiteturais para cada dilema (ex.: Loki vs ELK, Tempo vs Jaeger, Thanos vs Prometheus remote write)
2. Produzir matriz de trade-offs (custo, ops, facilidade, portabilidade)
3. Entregar síntese e recomendação para mesa técnica decidir

Saída esperada:
- Documento comparativo curto por tema (1 página)
- Checklist de critérios técnicos para decisão
